## 1. Executive Summary

This document plans the **rebranding of OllamaCX → ExecPrompt** and alignment with the **DayOfGeek.com** brand identity. The app retains its CyberTerm aesthetic and all existing functionality — the focus is on name, identity, iconography, and package ID changes.

| Decision | Value |

| **App Name** | ExecPrompt |
| **Brand / Publisher** | DayOfGeek.com |
| **Domain** | dayofgeek.com |
| **Android Package ID** | `com.dayofgeek.execprompt` |
| **Linux Application ID** | `com.dayofgeek.execprompt` |
| **Dart Package Name** | `execprompt` |
| **Design System** | CyberTerm (unchanged) |
| **Repository** | `dayofgeek` monorepo (app in `execprompt/`, website in `dayofgeek/`) |

### Guiding Principle

> "I have reservations that expanding the functionality too much and incorrectly might break the amazing experience we already have."

Rebranding is cosmetic — zero functional changes to the chat experience.

## 2. Branding Overview

### 2.1 ExecPrompt Identity

**ExecPrompt** — a premium Ollama client with a CyberTerm aesthetic. The name evokes "executive prompt" — commanding AI from a terminal interface.

**Brand Identity:**
- **Publisher:** DayOfGeek.com
- **Design Philosophy:** Cyberpunk Terminal / Retro-Futurism (unchanged)
- **Primary Font:** JetBrains Mono (unchanged)
- **Color Palettes:** P1 Green, P3 Amber, P4 White, Neon Cyan, Synthwave (unchanged)

### 2.2 App Icon Concept

The ExecPrompt icon should reflect the CyberTerm aesthetic:
- **Shape:** Terminal cursor / command prompt motif (`>_` or `▸_`)
- **Colors:** Phosphor green (#33FF33) on deep black (#0A0F0A) as the primary variant
- **Style:** Sharp edges, no rounded corners, monospace feel
- **Variants needed:**
  - Android adaptive icon (foreground + background layers)
  - Android legacy icon (48-192px mipmap PNGs)
  - Linux desktop icon (256x256 PNG or SVG)
  - Web favicon (for dayofgeek.com)
  - Play Store listing (512x512 high-res)

## 4. App Icon & Launcher Icon Plan

### 4.1 Current State

All Android launcher icons are **stock Flutter defaults** (the blue Flutter logo). No custom icon has ever been set. No Linux desktop icon exists.

### 4.2 Icon Design Requirements

**Concept:** Terminal prompt motif — the `>_` cursor symbol rendered in the CyberTerm phosphor style.

**Design spec:**
```
┌──────────────────┐
│                  │
│   >_             │  Phosphor green (#33FF33) prompt symbol
│                  │  on deep black (#0A0F0A) background
│                  │  JetBrains Mono or similar monospace
└──────────────────┘
```

**Variants:**
| Asset | Size | Format | Purpose |
|-------|------|--------|---------|
| `ic_launcher.png` (mdpi) | 48×48 | PNG | Android legacy |
| `ic_launcher.png` (hdpi) | 72×72 | PNG | Android legacy |
| `ic_launcher.png` (xhdpi) | 96×96 | PNG | Android legacy |
| `ic_launcher.png` (xxhdpi) | 144×144 | PNG | Android legacy |
| `ic_launcher.png` (xxxhdpi) | 192×192 | PNG | Android legacy |
| `ic_launcher_foreground.png` | 432×432 | PNG | Android adaptive (with 66dp safe zone) |
| `ic_launcher_background.xml` | — | XML drawable | Solid #0A0F0A background |
| `ic_launcher.xml` | — | XML | Adaptive icon definition |
| `ic_launcher_round.xml` | — | XML | Round adaptive icon |
| Desktop icon | 256×256 | PNG | Linux `.desktop` file |
| Play Store | 512×512 | PNG | Play Store listing |
| Favicon | 32×32 + 16×16 | ICO/PNG | dayofgeek.com |
| Web manifest icons | 192×192 + 512×512 | PNG | PWA/web manifest |

### 4.3 Icon Generation Approach

**Option A: `flutter_launcher_icons` package (Recommended)**

Add to `pubspec.yaml`:
```yaml
dev_dependencies:
  flutter_launcher_icons: ^0.14.3

flutter_launcher_icons:
  android: true
  ios: false
  image_path: "assets/icon/execprompt_icon.png"
  adaptive_icon_background: "#0A0F0A"
  adaptive_icon_foreground: "assets/icon/execprompt_icon_foreground.png"
  min_sdk_android: 21
```

Then run:
```bash
dart run flutter_launcher_icons
```

This auto-generates all Android mipmap sizes and adaptive icon XML from a single source image.

**Option B: Manual generation**

Use a tool like Android Asset Studio or Figma to export each size manually. More control but more maintenance.

### 4.4 Icon Source Asset Requirements

To use `flutter_launcher_icons`, we need:

1. **`assets/icon/execprompt_icon.png`** — 1024×1024 PNG, full icon with background (for legacy/fallback)
2. **`assets/icon/execprompt_icon_foreground.png`** — 432×432 PNG, just the `>_` symbol centered with padding (for adaptive icon foreground layer)

These source assets need to be **designed and created** — options:
- Design in Figma/Inkscape with JetBrains Mono glyphs
- Programmatically generate with a script (Canvas/SVG → PNG)
- Commission from a designer

## 6. Play Store & Package ID Considerations

### 6.1 Package ID Decision

The Android `applicationId` is the **permanent identity** of your app on the Play Store. It cannot be changed after first publish without creating a new listing.

**Recommendation:** `com.dayofgeek.execprompt`

This requires:
1. A Google Play Developer account ($25 one-time registration fee)
2. The account can be under any name/entity — "DayOfGeek" or personal
3. The `applicationId` in `build.gradle.kts` must match what you publish under

**Do this before building any release APKs** to avoid having to rebuild.

### 6.2 Signing Key

Android release builds require a signing key. This is separate from the Play Store account:
- Generate a keystore: `keytool -genkey -v -keystore execprompt-release.jks -keyalg RSA -keysize 2048 -validity 10000 -alias execprompt`
- Store securely — losing this key means you can never update the app on the Play Store
- Configure in `android/key.properties` (NOT committed to git)

### 6.3 Migration Concern

Changing from `com.zervin.ollamacx` to `com.dayofgeek.execprompt`:
- If already published under the old ID → this is a **new app listing** (users won't get auto-updates)
- If not yet published → no concern, just use the new ID from the start
- The old `com.zervin` namespace should be completely removed
